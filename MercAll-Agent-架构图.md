# MercAll Agent 对话能力架构图

## 1. 整体架构概览

```mermaid
graph TB
    subgraph 开发者/用户层
        A[开发者] -->|API调用| B[MerCall API]
        C[用户] -->|访问网站| D[Mercall 首页]
        E[用户] -->|下载安装| F[ClawLite 客户端]
    end

    subgraph Mercall 服务层
        B -->|统一计费/限流| G[MerCall 后端]
        D -->|对话请求| G
        G -->|调用| H[协议转换服务]
    end

    subgraph 对话引擎层
        H --> I[Hermes]
        H --> J[Claude Code]
        H --> K[自研 Claw]
    end

    subgraph ClawLite 部署层
        F -->|API调用| G
        L[用户本地] -->|运行| M[Claw/Hermes]
        F -->|控制| M
    end
```

## 2. Api 对话流程 (开发者模式)

```mermaid
sequenceDiagram
    participant D as 开发者
    participant MC as Mercall API
    participant NA as NewAPI协议转换
    participant BE as 后端引擎

    D->>MC: 1. 申请 API Key
    MC->>D: 2. 返回 API Key
    D->>MC: 3. 调用对话接口 + API Key
    MC->>MC: 4. 统一计费/限流/日志
    MC->>NA: 5. 请求协议转换
    NA->>BE: 6. 调用后端引擎
    BE->>NA: 7. 返回结果
    NA->>MC: 8. 协议转换
    MC->>D: 9. 返回对话结果
```

## 3. 首页 Agent 对话流程

### 3.1 在线网页对话模式

```mermaid
sequenceDiagram
    participant U as 用户
    participant W as Mercall 网站
    participant MB as Mercall 后端
    participant HE as Hermes
    participant CC as Claude Code
    participant CL as Claw

    U->>W: 访问 Mercall 首页
    U->>W: 输入对话内容
    W->>MB: 发送对话请求
    MB->>MB: 1. 计费/限流/日志
    MB->>HE: 2. 调用 Hermes
    MB->>CC: 3. 调用 Claude Code
    MB->>CL: 4. 调用自研 Claw
    HE-->>MB: 返回结果
    CC-->>MB: 返回结果
    CL-->>MB: 返回结果
    MB-->>W: 统一返回
    W-->>U: 显示对话结果
```

## 4. 接口协议规范 (LeapAgentApiSdk)

```mermaid
graph LR
    subgraph 统一接口协议 LeapAgentApiSdk
        A[Claw] -->|HTTP| D[MerCall 统一接入层]
        B[Claude Code] -->|HTTP| D
        C[Hermes] -->|HTTP| D
        E[ClawLite] -->|HTTP| D
    end

    D --> F[可切换任意后端]
    F --> G[后端1]
    F --> H[后端2]
    F --> I[后端N]
```
